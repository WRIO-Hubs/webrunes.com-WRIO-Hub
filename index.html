Web Evolution

Presentation

Semantic Protocol

Structured data is a standardized Linked Data  format, i.e. pages may use each other's data and come together like a puzzle or Lego blocks. Structured data means a key:value format that features machine readability. For example, on a recipe page these would be the ingredients, the cooking time and temperature, the calories, and so on. Such pages are cross-domain & virus free, enable automatic data processing and pave the way for the development of a distributed and autocurated web. The Semantic Protocol is the Web 3.0 language that ensures free and continuous data exchange between websites and apps of the new web. Create  your first Web 3.0 page.

User Interface

Pages are no longer stand-alone and are interlinked, while data is no longer restricted to specific website boundaries and is rendered into a page in a Theme selected by yourself, determining your user interface. Such an approach is known as user-centric and perfectly matches the taste and physical abilities of a user, as well as technical characteristics of the device used for browsing.

Functionality

Just like with the user interface, functions of pages are user-centric: in the same way you customize your browser and mobile functionality, the browsing experience is determined by the set of Web 3.0 apps you select, rather than by settings and technical capacities of a website. Web 3.0 apps are cloud apps and can operate on the most simple devices; they can be installed in one click; they are always up-to-date, do not need to download or setup any software. All resource consuming operations are performed on the service provider’s servers, displaying the end result on the user’s screen.

Metadatabase

A metadatabase of Web 3.0 pages and users' assets repository where blockchain address functions as ID. Access is granted through the use of a cryptokey stored on the user’s side. Metadata is used to search for and correlate content with the user profile. Automatic correlation, classification, and cataloging of web data against users’ interest matrices will allow the creation of predictive user-centric services and search engines of new generation.

Internet OS

An Internet OS is an operating system designed to run all of its applications and services through a web browser. The advantages of such an OS is that it runs on a thin client, allowing cheaper, more easily manageable computer systems. All applications are designed on the basis of cross-platform and open standards; and do not tie user's applications, documents, and preferences to a single computer, but rather place them in the Internet cloud. The Internet OS has also been promoted as the perfect type of platform for software as a service. The platform for the blockchain-powered Web 3.0 apps, serving as an upgrade to a distributed Web 3.0 ­— the future of the browsing experience.

Why is WRIO Internet OS important?

The Web 1.0 was simply a huge catalog of documents. It evolved into the Web 2.0: the web of social media and search engines, but still remains unsecure and centralized with manual data processing. The Web 3.0 is a secure web of robots and automatic data processing. While the Web 1.0 and Web 2.0 had websites with a core idea, the Web 3.0 has a user in it. Find more at the White Paper  chapter: WRIO Internet OS — All you need is a browser.

Who can use WRIO Internet OS, and why?

WRIO Internet OS was created to be an alternative to today’s Web 2.0 which is cluttered, non-secure, and with large scale privacy breaches almost daily in the news. The purpose of this project is for everyone to benefit from a user-centric browsing experience utilizing the semantic web with automatic data processing and storing metadata through blockchain due to the large degree of security that it yields. Authors, Entrepreneurs, Developers, and Users will all be incentivized by WRIO’s user-centric services and DApps ecosystem.

What is the current state of the WRIO Internet OS project?

The MVP was launched in April 2018 and we’re on schedule (see Roadmap) to further develop the project to the point where we can soon bring a generation of fully distributed and secure Web 3.0 browsing.

How does WRIO Internet OS utilize the blockchain?

Blockchain technology will be utilized to store the metadata of each generated page (in Linked Data  format) prior to being integrated to the user’s Interest Matrix  compiled in their profile. It helps in the auto process of linking decentralized pages through a global metadatabase.

Why are you using Ethereum? 

Ethereum has matured dramatically over the past few years. Because of its maturity and therefore its blockchain security, as well as its scaling capability plans, we believe it’s the best option to run this platform on.

If you’re using Ethereum and storing metadata on it, won’t it bloat the network? 

The simple answer to that is no. Due to the semantic infrastructure of WRIO Internet OS, the metadata will equal ‘tags’ on the blockchain, and these tags will be limited to <64 bytes, and every post on the platform, for instance, has a “wallet” attached to it with a list of transactions and metadata (tags). To put this into perspective, Wikipedia has about 5 million articles and the whole ETH transaction capacity is about .5 million per day. That means the whole wiki metadata is equivalent to 10 days of Ethereum capacity only.

The name originates from the acronym of “webRunes Input/Output Internet Operating System ”. Its browser-driven approach which allows the system to work with any device with a browser and Internet connection, which is reflected in the project motto: “All you need is a browser”.

What’s the difference between WGD and CRD tokens?

WGD  will be traded as a stake in the platform and will be traded on exchanges once the Token Sale concludes. CRD  (Credits) are in-platform tokens to be used as Credit inside the platform, such as upvotes to authors, donations to users, or content upvotes for a better user experience (see Interest Matrix  algorithm).
